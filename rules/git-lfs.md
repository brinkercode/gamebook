# Git + Git LFS Rules

## Rationale

Unreal Engine projects contain binary assets (textures, meshes, audio, compiled shaders) that Git cannot diff or merge. Git LFS tracks these large files by pointer while storing the actual data on an LFS server. Without LFS, a single commit adding 4K textures can make the repo permanently multi-gigabyte.

## .gitattributes (canonical pattern)

Copy this exactly. Every Unreal binary format must be listed.

```gitattributes
# Unreal Engine binary assets — store via Git LFS
*.uasset filter=lfs diff=lfs merge=lfs -text
*.umap   filter=lfs diff=lfs merge=lfs -text
*.uplugin binary
*.uproject binary

# Compiled shaders
*.ushaderbytecode filter=lfs diff=lfs merge=lfs -text
*.ushaderlib      filter=lfs diff=lfs merge=lfs -text

# Audio
*.wav  filter=lfs diff=lfs merge=lfs -text
*.mp3  filter=lfs diff=lfs merge=lfs -text
*.ogg  filter=lfs diff=lfs merge=lfs -text
*.wem  filter=lfs diff=lfs merge=lfs -text   # Wwise encoded audio

# Texture source (raw)
*.psd  filter=lfs diff=lfs merge=lfs -text
*.tga  filter=lfs diff=lfs merge=lfs -text
*.png  filter=lfs diff=lfs merge=lfs -text
*.jpg  filter=lfs diff=lfs merge=lfs -text
*.jpeg filter=lfs diff=lfs merge=lfs -text
*.exr  filter=lfs diff=lfs merge=lfs -text
*.hdr  filter=lfs diff=lfs merge=lfs -text
*.bmp  filter=lfs diff=lfs merge=lfs -text

# 3D model source
*.fbx  filter=lfs diff=lfs merge=lfs -text
*.obj  filter=lfs diff=lfs merge=lfs -text
*.abc  filter=lfs diff=lfs merge=lfs -text

# Video
*.mp4  filter=lfs diff=lfs merge=lfs -text
*.mov  filter=lfs diff=lfs merge=lfs -text
*.avi  filter=lfs diff=lfs merge=lfs -text

# Wwise project files
*.wwu  filter=lfs diff=lfs merge=lfs -text
*.wproj filter=lfs diff=lfs merge=lfs -text

# Archives
*.zip  filter=lfs diff=lfs merge=lfs -text
*.7z   filter=lfs diff=lfs merge=lfs -text

# Precompiled libraries
*.lib  filter=lfs diff=lfs merge=lfs -text
*.dll  filter=lfs diff=lfs merge=lfs -text
*.so   filter=lfs diff=lfs merge=lfs -text
*.dylib filter=lfs diff=lfs merge=lfs -text
*.a    filter=lfs diff=lfs merge=lfs -text

# Force text for all source and config
*.h    text eol=lf
*.cpp  text eol=lf
*.cs   text eol=lf
*.ini  text eol=lf
*.json text eol=lf
*.md   text eol=lf
*.sh   text eol=lf
```

- **Never remove a pattern from this list** — missing patterns create non-LFS binaries in the commit history that can never be removed without a history rewrite.
- **Add new binary extensions before committing the asset** — check the `.gitattributes` pattern before adding any new file type.
- **`eol=lf` on all text files** — prevents CRLF line-ending conflicts between Windows and Linux/macOS builds.

## What to never commit

```
# .gitignore — required entries for Unreal Engine projects
Saved/
Intermediate/
DerivedDataCache/
Build/
Binaries/
.vs/
*.suo
*.sln        # Regenerated by UBT — don't track
*.vcxproj    # Same
*.generated.h  # Generated by UHT — don't track
SourceArtist/ # If present — raw art source before import
```

- **`Saved/`** — contains logs, crash dumps, screenshots, editor config. Never commit.
- **`Intermediate/`** — UBT build cache. Committing it causes build conflicts when build machine differs.
- **`DerivedDataCache/`** — cooked shader and asset cache. Multiple gigabytes; project-local only.
- **`Binaries/`** — compiled game binaries. Never commit; rebuild from source.

## LFS lock workflow (when team size > 2)

Binary Unreal assets cannot be merged — two people editing the same `.uasset` simultaneously will have a conflict that requires one person to throw away their work. Use LFS locking to prevent this:

```bash
# Before editing a binary asset, acquire a lock
git lfs lock Content/Weapons/SK_Rifle.uasset

# See who has locks
git lfs locks

# When done and committed, release
git lfs unlock Content/Weapons/SK_Rifle.uasset
```

- **Lock before opening an asset in the editor** — not after. Locks are advisory; Git won't stop you from editing without one, but your team will be unhappy.
- **Unlock immediately after committing** — long-held locks block the team. If a lock is stale (person is gone), repo admin can force-unlock: `git lfs unlock --force <path>`.
- **Source art files (`.psd`, `.fbx`, `.ai`) require locks too** — even if you plan to re-export, the commit that adds the new binary is still a conflict risk.

## Max binary size per commit

| Asset category | Max size per file | Notes |
|---|---|---|
| Texture (`.uasset` imported) | 32 MB | 4K BC-compressed; 8K in exceptional cases |
| Skeletal mesh + animations | 50 MB per package | Split skeleton from AnimSequences if larger |
| Static mesh | 10 MB | Pre-LOD high-res mesh |
| Level (`.umap`) | 100 MB | Larger levels should use Level Streaming sub-levels |
| Wwise SoundBank | 200 MB | Per-bank; split by context if larger |
| Source texture (`.psd`, `.png`) | 64 MB | Raw art before UE import |

- **Never commit a single binary over 200 MB** — LFS servers have per-file quotas and large files slow checkout.
- **Split large levels into streaming sub-levels** — a monolithic 500 MB `.umap` is not acceptable. Use `LT_` sub-level assets (see [ue5-naming.md](ue5-naming.md)).

## Pulling and hydrating LFS assets

```bash
# Standard clone — fetches LFS pointers but not data
git clone <repo>

# Hydrate all LFS pointers (full local copy)
git lfs pull

# After a git pull, fetch new LFS objects
git lfs fetch && git lfs checkout

# Check which files are LFS pointers (not hydrated)
git lfs status
```

- **Run `git lfs pull` after every `git pull` in CI** — CI runners often start with a shallow clone; LFS objects are not fetched automatically.
- **`git lfs migrate import`** to retroactively LFS-track a file that was committed without LFS — this rewrites history. Coordinate with the team before running it.

## Branch and merge strategy

- **Feature branches for asset work, same as code** — `feature/weapon-rifle-lod`, `feature/zone-01-audio`. Merge via PR.
- **Never rebase commits containing LFS objects** — rebasing recomputes commit hashes; LFS pointers in rebased commits can point to objects not uploaded to the remote. Use merge commits for LFS-heavy branches.
- **Squash only when all LFS objects are already pushed** — verify with `git lfs push --all origin` before squashing.
