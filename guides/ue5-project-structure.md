# UE5 Project Structure вЂ” Source / Content / Config / Plugins

> Reference for what `gamebook-init.sh` produces. Every folder and file has an explicit purpose. The layout enforces module boundaries between game code, editor code, and test code, and matches the Content conventions in [asset-pipeline.md](asset-pipeline.md). Agents read this file before scaffolding any new subsystem.

---

## Bootstrap

```bash
mkdir MyGame && cd MyGame
bash ~/.claude/gamebook/scripts/gamebook-init.sh \
    --name "MyGame" --desc "FPS vertical slice" --ue-root "/home/dev/UnrealEngine"
```

Pass `--multiplayer` to scaffold the dedicated server target and Replication Graph stub. Pass `--no-steam` for EGS-only builds.

## Module Rules

### MyGame (Runtime)

```
Dependencies in MyGame.Build.cs:
PublicDependencyModuleNames:
    - Core, CoreUObject, Engine, InputCore
    - EnhancedInput
    - GameplayAbilities, GameplayTags, GameplayTasks
    - CommonUI
    - Niagara
    - OnlineSubsystem, OnlineSubsystemSteam, OnlineSubsystemEOS

PrivateDependencyModuleNames:
    - HTTP, Json, JsonUtilities     в†ђ for Steam/EOS API calls
    - UMG
    - Slate, SlateCore
```

The game module must not reference `MyGameEditor` вЂ” that module exists to add editor-only code and imports `UnrealEd`. Including editor code in the runtime module breaks shipping builds.

### MyGameEditor (Editor-only)

```
Type = Editor
LoadingPhase = PostEngineInit

Depends on: MyGame, UnrealEd, Blutility, EditorFramework
```

Put asset factory classes, Blueprint node customizations, and editor utility widgets here. This module is stripped from shipping builds.

### MyGameTests (Test)

```
Type = RuntimeAndProgram

Depends on: MyGame, FunctionalTesting, AutomationController
```

The Gauntlet runner loads this module in `Test` build config. Functional Test actors in `Content/Tests/Maps/` are Blueprint actors that reference C++ test helpers from this module.

## Content Browser Folder Rules

- **`Core/`** вЂ” shared, low-level assets with no level-specific context. Master materials, GAS data, global data tables.
- **`Characters/` / `Weapons/` / `Environment/`** вЂ” organized by subject, not by asset type. All meshes, materials, and animations for a subject live together.
- **`Levels/`** вЂ” maps and their streaming sub-levels. Sub-levels are `LT_` prefixed.
- **`UI/`** вЂ” all UMG widgets, styles, and Common UI assets. No game-logic Blueprints.
- **`VFX/`** вЂ” all Niagara systems and emitters. Weapon VFX lives under `VFX/Combat/`, not under `Weapons/`.
- **`Tests/`** вЂ” Functional Test maps and test actor instances. Never ship. Set `bSkipEditorContent=True` in DefaultGame.ini to exclude from cook.

Never place assets in `Content/` root. The Content Browser root level should contain only these named folders.

---

## Why a Generator

- **Token-cheap** вЂ” agents `cat` real files from the scaffolded project instead of inferring structure from prose.
- **Diffable** вЂ” Makefile improvements are versioned in the gamebook, not in individual project docs.
- **Deterministic** вЂ” every project gets the same `make gate` contract: `smoke в†’ automation-critical в†’ (build-server if applicable)`.
- **Module boundaries enforced** вЂ” `MyGameEditor` and `MyGameTests` are separate modules; the toolchain prevents accidental shipping of editor code.

---

## Customizing Per-Project

Templates use `{{PROJECT_NAME}}`, `{{DESCRIPTION}}`, `{{UE_ROOT}}`. `gamebook-init.sh` substitutes from CLI flags or interactive prompts. Edit a template under `gamebook/templates/` to change every future project. For per-project deviations, edit the file after init вЂ” `gamebook-init.sh` never overwrites existing files.

---

## Key Rules

1. **Three modules, always** вЂ” `MyGame` (runtime), `MyGameEditor` (editor-only), `MyGameTests` (tests). Never add game logic to editor module, never ship test code.
2. **Content folder = subject, not asset type** вЂ” `Characters/Hero/Meshes/` not `Meshes/Characters/Hero/`.
3. **C++ only in `Source/`** вЂ” no `.cpp` or `.h` in `Content/`. Blueprint logic in `Content/Blueprints/` or alongside the subject it governs.
4. **All binary assets via Git LFS** вЂ” `.gitattributes` committed before any `.uasset`.
5. **`Saved/` is fully gitignored** вЂ” logs, cooked content, and staged builds never committed.
6. **Naming conventions from [asset-pipeline.md](asset-pipeline.md) are enforced on import** вЂ” rename in the editor, not via OS, to preserve redirectors.
7. **`Content/Tests/` excluded from shipping cook** вЂ” `bSkipEditorContent=True` covers editor modules; also add `+DirectoriesToNeverCook=(Path="/Game/Tests")` to DefaultGame.ini.
